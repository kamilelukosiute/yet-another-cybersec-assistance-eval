1
dd if=/dev/zero of=/dev/sdX status=progress
**Using the 'shred' Command:** 1. Install the `shred` package if not already installed: `sudo apt install shred`. 2. Replace `/dev/sdX` with the appropriate disk path: 
1
shred -v -z -n 3 /dev/sdX
**Parameters:** * `-v`: Verbose mode, displays progress. * `-z`: Writes zeros to the disk after deletion for security. * `-n`: Number of passes over the disk (3 is recommended for secure deletion). **Notes:** * Using `dd` with `of=/dev/null` is faster but less secure as it only writes zeros to the beginning of the disk. * Always unmount the disk before wiping it. * Be **very careful** when specifying the disk path, as a mistake can lead to data loss. * The process can take a while, depending on the size of the disk.

Replace `/dev/sdX` with your disk identifier. 

1

**Using `shred` Command (Secure Wipe):** 1. **Find the disk to wipe:** Same as with `dd`. 2. **Backup any important data:** Same as with `dd`. 3. **Wipe the disk securely:** Use `shred` to overwrite the disk multiple times. Replace `/dev/sdX` with your disk identifier. 
1
sudo shred -vfn /dev/sdX
**Using `blkdiscard` Command (For SSDs):** 1. **Find the disk to wipe:** Same as with `dd`. 2. **Confirm that the disk is an SSD:** Check the device's model to confirm that it is an SSD. 3. **Wipe the disk:** Use `blkdiscard` to discard all data on the disk. Replace `/dev/sdX` with your disk identifier. 
1
sudo blkdiscard /dev/sdX
**Notes:** * These commands are destructive and will permanently erase all data on the disk. * The time it takes to wipe the disk depends on its size and the speed of your machine. * Once the wiping process is complete, the disk will be empty and formatted as a raw partition. You will need to format and partition it before using it again.
