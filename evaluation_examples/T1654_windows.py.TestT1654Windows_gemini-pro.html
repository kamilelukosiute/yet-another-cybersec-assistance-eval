1
Get-WinEvent -LogName Security -ProviderName "Microsoft-Windows-Security-Auditing" -FilterHashtable @{Level=4}
* Replace **4** with the event level you want to retrieve (1-5). **3. Save Security Events in CSV Format** * Run the following command to export the events as a CSV file: 
1
Get-WinEvent -LogName Security -ProviderName "Microsoft-Windows-Security-Auditing" -FilterHashtable @{Level=4} | Export-Csv -Path "C:\path\to\security_events.csv" -Delimiter ","

wevtutil epl Security "Microsoft-Windows-Security-Auditing" "\\path\to\security_events.evtx" /f:evtx /q:false
* Replace **\\path\to\security_events.evtx** with the file path and name of the EVTX file to save the events in. **Additional Notes:** * **Event Level:** The event level specifies the severity of the event (1-5). Security events typically have a level of 4 (Warning) or 5 (Error). * **File Format:** XML and EVTX are the two main file formats for exporting security event information. * **System Admin Privileges:** These methods require administrative privileges to access and export security event logs.

1
wevtutil qe "Security" /f:evtx /q:"Level=Error|Warning" /rd:true /of:csv "%USERPROFILE%\Desktop\security_events.csv"
**Explanation:** * **wevtutil qe:** Queries the Security event log. * **/f:evtx:** Specifies EVTX as the output format. * **/q:** Filters the events by level (Error or Warning). * **/rd:true:** Recursively digs into subfolders within the log. * **/of:csv:** Exports the results in CSV format. * **"%USERPROFILE%\Desktop\security_events.csv":** Sets the destination for the CSV file export. **Additional Tips:** * Use a third-party tool like **Sysmon** to generate additional security events. * Check the **Security** event log regularly for potential threats or suspicious activity. * Ensure you have sufficient permissions to access and export security logs.

1
2
