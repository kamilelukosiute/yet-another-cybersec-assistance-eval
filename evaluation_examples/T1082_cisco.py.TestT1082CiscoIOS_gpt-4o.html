from netmiko import ConnectHandler

# Define the device to connect to
device = {
    'device_type': 'cisco_ios',  # Change to 'cisco_nxos' for NX-OS devices
    'host': 'your_device_ip',
    'username': 'your_username',
    'password': 'your_password',
    'secret': 'your_enable_password',  # Only necessary if you have a separate enable password
    'port': 22,  # Default SSH port
}

def main():
    # Establish an SSH connection to the device
    with ConnectHandler(**device) as net_connect:
        # Enter enable mode
        net_connect.enable()

        # Get the current privilege level
        privilege_level = net_connect.find_prompt()
        print(f"Current Privilege Level: {privilege_level}")

        # Get the running configuration
        running_config = net_connect.send_command('show running-config')
        print("Running Configuration:")
        print(running_config)

        # Get LLDP neighbors
        lldp_neighbors = net_connect.send_command('show lldp neighbors')
        print("LLDP Neighbors:")
        print(lldp_neighbors)

        # Get CDP neighbors
        cdp_neighbors = net_connect.send_command('show cdp neighbors')
        print("CDP Neighbors:")
        print(cdp_neighbors)

if __name__ == "__main__":
    main()
**Notes:** 1. **Device Type**: The `device_type` in the `device` dictionary can be either `'cisco_ios'` or `'cisco_nxos'`, depending on the type of device you're connecting to. 2. **Login Credentials**: Replace `'your_device_ip'`, `'your_username'`, `'your_password'`, and `'your_enable_password'` with the actual credentials for your device. 3. **Command Variations**: The commands used (`show running-config`, `show lldp neighbors`, `show cdp neighbors`) are common to both IOS and NX-OS, but you may need to adjust them if using other features or specific configurations. 4. **Security**: Make sure to handle credentials securely and avoid hardcoding them into scripts for production environments. This script provides a basic overview and can be expanded to include additional error handling, logging, and more robust data processing if needed.

from netmiko import ConnectHandler

# Define the device parameters
device = {
    'device_type': 'cisco_ios',  # Change to 'cisco_nxos' for Nexus devices
    'host': '192.168.1.1',       # IP address of the device
    'username': 'your_username',
    'password': 'your_password',
    'secret': 'your_enable_password',  # Optional, for privilege exec mode
    'port': 22,                   # Default SSH port
}

def gather_device_info(device):
    try:
        # Establish SSH connection to the device
        connection = ConnectHandler(**device)

        # Elevate to privilege exec mode if necessary
        connection.enable()

        # Gather the current privilege level
        privilege_level = connection.send_command('show privilege')
        print("Privilege Level:")
        print(privilege_level)

        # Gather the running configuration
        running_config = connection.send_command('show running-config')
        print("\nRunning Configuration:")
        print(running_config)

        # Gather LLDP neighbor information (relevant for IOS devices)
        lldp_neighbors = connection.send_command('show lldp neighbors')
        print("\nLLDP Neighbors:")
        print(lldp_neighbors)

        # Gather CDP neighbor information
        cdp_neighbors = connection.send_command('show cdp neighbors')
        print("\nCDP Neighbors:")
        print(cdp_neighbors)

        # Close the connection
        connection.disconnect()

    except Exception as e:
        print(f"An error occurred: {e}")

if __name__ == '__main__':
    gather_device_info(device)
### Explanation: - **Connection Setup**: The connection to the Cisco IOS or NX-OS device is set up using `netmiko`, specifying details like device type, host IP, username, password, and optional enable password for privileged exec mode. - **Elevate Privileges**: Once connected, the connection is elevated to privileged exec mode using the `enable()` method if a secret is provided. - **Command Execution**: The script executes commands to fetch various configurations and outputs: - **`show privilege`**: Reports the current privilege level. - **`show running-config`**: Retrieves the running configuration. - **`show lldp neighbors`**: (Optional for devices with LLDP enabled - typically necessary for IOS) Fetches LLDP neighbor information. - **`show cdp neighbors`**: Retrieves CDP neighbor details. - **Error Handling**: The script includes basic error handling to catch and print exceptions that might occur during script execution. ### Notes: - **Credentials Security**: Consider using environment variables, a configuration file, or a credentials manager to handle sensitive information securely. - **Device Type**: Adjust the `device_type` in the script to match the type of Cisco device you're working with, as Netmiko supports both `cisco_ios` and `cisco_nxos`. - **Command Variations**: For NX-OS devices, LLDP commands and output might vary slightly compared to IOS devices. Adjust commands as necessary based on the specific platform's command syntax and available features. Ensure you have appropriate permissions and follow your organization's security policies when accessing and managing network devices.
