Function Add-ScriptToStartup {
    Param (
        [string]$ScriptFile,
        [string]$TargetFolder = $env:USERPROFILE + "\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup"
    )
    
    if (-not (Test-Path $ScriptFile)){
        Write-Error "$ScriptFile does not exist."
        return
    }

    if (-not (Test-Path $TargetFolder)){
        New-Item -Path $TargetFolder -ItemType Directory
    }

    Copy-Item -Path $ScriptFile -Destination $TargetFolder
    
    Write-Host "Script added to Startup."
}
To use it, open a PowerShell window with elevated privileges and run the following command: 
1
Add-ScriptToStartup -ScriptFile "C:\Users\John\Scripts\MyScript.ps1"
**Additional Considerations** * Ensure that the script has the necessary permissions to run on user login. * Test the script thoroughly before deploying it in a production environment. * Consider using a log file to track any errors or issues with script execution.

## Automating Script Execution at Windows Login ### Method 1: Run Script from Startup Folder 1. **Create a shortcut to the script:** Right-click your script file and select "Send to" > "Desktop (create shortcut)." 2. **Copy the shortcut to the Startup folder:** - Press **Win+R** to open the Run dialog box. - Type `shell:startup` and press Enter. - Paste the shortcut into the Startup folder. ### Method 2: Use Task Scheduler 1. **Create a new task:** Open the Task Scheduler by searching for it in the Start menu. 2. **Configure task settings:** - **Name:** Give the task a descriptive name. - **Trigger:** Select "At log on" for the trigger type. - **Action:** Click "New" and select "Start a program." Browse to your script file. 3. **Configure user rights:** - Go to the "General" tab. - Under "User account," select the user account(s) you want the script to run for. - Check the "Run with highest privileges" box if necessary. ### Method 3: Use Group Policy (Windows Pro and Enterprise only) 1. **Open Group Policy Editor:** Press **Win+R** and type `gpedit.msc`. 2. **Navigate to the script policy:** - For computer-wide scripts: Computer Configuration > Administrative Templates > System > Scripts (Startup/Logoff) - For user-specific scripts: User Configuration > Administrative Templates > System > Scripts (Startup/Logoff) 3. **Configure script settings:** - **Script:** Browse to your script file. - **Enabled:** Select "Enabled." - **Set any additional options:** Such as "Run only when user logs on to a domain" or "Run whether or not the user is logged on." ### Automating Process with PowerShell Scripts You can automate the above methods using PowerShell scripts: #### Method 1 (Startup Folder):
