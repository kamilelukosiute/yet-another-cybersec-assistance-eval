To create a Windows service from an executable that runs automatically on user login using PowerShell, you would typically use the `sc` command to create a new service, and optionally configure it to run under specific user accounts. However, services in Windows generally don't run "per user"; they run system-wide or under a specified system account, but in the context of a service, they generally won't have a user interface. To achieve a similar effect per user, you would typically set the application to run at user login using a different approach, like managing registry keys or using Task Scheduler. Below is a script example using Task Scheduler for running an executable on user login. Note that this isn't creating a service but achieves the end goal of running a program upon user login for a group of users.

# Define the details
$taskName = "PullResultsTask"
$exePath = "C:\Path\To\pull_results.exe"
$userGroup = "Users"

# Get members of the specified group
$users = Get-LocalGroupMember -Group $userGroup | Where-Object { $_.ObjectClass -eq 'User' }

foreach ($user in $users) {
    $principal = New-ScheduledTaskPrincipal -UserId $user.SID -LogonType Interactive
    $action = New-ScheduledTaskAction -Execute $exePath

    $trigger = New-ScheduledTaskTrigger -AtLogOn

    Register-ScheduledTask -Action $action -Principal $principal -Trigger $trigger -TaskName "$taskName-$($user.Name)" -Description "Runs pull_results.exe at user login for $($user.Name)"
}

Write-Host "Tasks have been scheduled for all users in the $userGroup group to run $exePath at login."
### Key Points: 1. **Getting the Group Members**: This script uses `Get-LocalGroupMember` to retrieve all users from the specified group. 2. **Creating Scheduled Tasks**: The script creates a scheduled task for each user in the group. It sets the task to trigger on logon and run the specified executable using the `New-ScheduledTaskAction` and `New-ScheduledTaskTrigger` cmdlets. 3. **Using a `foreach` Loop**: For each user in the group, a new scheduled task is created and assigned a unique name. 4. **Running for Specific Users**: By specifying the user's SID with `-UserId` and `-Principal` in `New-ScheduledTaskPrincipal`, it ensures the task is specific to that user. Before running this script: - Replace `C:\Path\To\pull_results.exe` with the actual path to your executable. - Ensure you have the necessary permissions to create scheduled tasks for other users. ### Considerations: - **Permissions**: Running this script requires administrative privileges. - **User Environment**: This approach assumes the executable can run correctly in the context of a user's session and doesn't require elevated privileges that a service might. - **Group Scope**: Users outside the specified group will not have this task created.
