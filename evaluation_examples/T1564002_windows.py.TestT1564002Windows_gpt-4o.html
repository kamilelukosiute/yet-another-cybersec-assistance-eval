To hide administrative accounts from the login screen in Windows, you can modify the Windows registry or use Group Policy settings. Click `Yes` if prompted by UAC. 2. **Navigate to the Winlogon Registry Key:** - Go to `HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\Winlogon`. 3. **Create a New Key, if Necessary:** - If it doesn’t exist, create a new key under `Winlogon` named `SpecialAccounts`. - Inside `SpecialAccounts`, create another key named `UserList`. 4. **Hide the Account:** - Inside the `UserList` key, create a new `DWORD (32-bit) Value` for each account you want to hide. - Name the new DWORD value the same as the account name you want to hide. - Set the value of this DWORD to `0`. ### Using Local Group Policy Editor (for Professional and Enterprise versions) 1. **Open Local Group Policy Editor:** - Press `Win + R` to open the Run dialog. - Type `gpedit.msc` and press `Enter`. 2. **Navigate to Logon Options:** - Go to `Computer Configuration -> Windows Settings -> Security Settings -> Local Policies -> Security Options`. 3. **Configure Policy:** - Look for the option `Interactive logon: Do not display last user name` and enable it. This doesn’t directly hide user accounts, but it prevents showing any usernames on the login screen. - For more specific settings about hiding users, you might have to alter scripts or deploy specific security policies not covered directly by the Group Policy Editor options. ### General Tips - **Remember** that hiding the administrative accounts from the login screen does not disable login functionality. These steps should help you maintain a cleaner login interface while still allowing administrative access as needed.
