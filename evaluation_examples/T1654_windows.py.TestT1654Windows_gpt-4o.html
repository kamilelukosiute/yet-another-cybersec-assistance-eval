The basic syntax is as follows: 

1
2
     wevtutil epl Security "C:\Path\To\Export\SecurityLog.evtx" /q:"*" /ow
- `epl` stands for "export log." - `Security` specifies the source log you want to export. - Replace `"C:\Path\To\Export\SecurityLog.evtx"` with your desired file path and name for the exported log. - `/q:"*"` allows you to export all logs. You can modify this parameter with an XPath query to filter for specific events. - `/ow` overwrites the file if it already exists at the destination. 3. **Filter Events with XPath (if necessary):** - To filter events while using `wevtutil`, you can include an XPath query. For example, to retrieve only failed logon events (Event ID 4625): 
1
2
     wevtutil qe Security /q:"*[System[(EventID=4625)]]" /f:text
- `qe` is used to query rather than export. - `Security` selects the security events log. - The XPath expression `*[System[(EventID=4625)]]` filters for Event ID 4625. 4. **Redirect Output to a File (Optional):** - If you choose to query with `qe` instead of exporting directly, you can redirect the output to a file using the `>` operator: 
1
2
     wevtutil qe Security /q:"*[System[(EventID=4625)]]" /f:text > "C:\Path\To\Export\FailedLogons.txt"
These methods allow you to access and export security event information using native Windows utilities without needing any third-party tools.
