Tell me how to wipe a disk from the command line on a macOS machine

Output

**Method 1: Using the `diskutil` Command** * Open Terminal (`Applications` > `Utilities` > `Terminal`). * Identify the disk you want to wipe using the `diskutil list` command. * Note the **disk identifier** (e.g., `/dev/disk1`). * Run the following command, replacing `/dev/disk1` with the disk identifier: 

1
sudo diskutil eraseDisk JHFS+ my-disk /dev/disk1
* Enter your password when prompted. **Method 2: Using the `dd` Command** * **Warning:** This method completely overwrites the disk, making data recovery impossible. * Open Terminal. * Identify the disk you want to wipe using the `diskutil list` command. * Note the **disk identifier** (e.g., `/dev/disk1`). * Run the following command, replacing `/dev/sdX` with the disk identifier: 
1
sudo dd if=/dev/zero of=/dev/disk1 bs=1M
* This command writes zeros to the entire disk, overwriting any existing data. **Method 3: Using a Secure Wipe Algorithm (Optional)** * Open Terminal. * Identify the disk you want to wipe using the `diskutil list` command. * Note the **disk identifier** (e.g., `/dev/disk1`). * Run the following command, replacing `/dev/disk1` with the disk identifier and the algorithm you want to use (e.g., `1-pass`): 
1
sudo diskutil secureErase -type 1-pass /dev/disk1
* Choose from the following algorithms: * `1-pass`: Single pass overwrite with zeros. * `2-pass`: Two pass overwrite with zeros and a random pattern. * `3-pass`: Three pass overwrite with a random pattern, zeros, and a random pattern. * `7-pass`: Gutmann Wipe algorithm (35 passes). **Note:** * Before wiping a disk, ensure that you have backed up all important data. * Wiping a disk is a permanent operation and cannot be undone. * Choose the secure wipe algorithm carefully, as some algorithms take significantly longer to complete.

1

1
2

* "HFS+J" is the file system type (Mac Extended). * "Volume Name" is the name of the volume to create. * `<disk_device_identifier>` is the device identifier of the mounted disk image. 5. Detach the disk image:

2
   hdiutil detach <disk_image_file>

Output

**Using the `diskutil` Command:** 1. Open Terminal. 2. Identify the disk you want to wipe using the command: 

1
diskutil list
3. Note the "Identifier" for the disk you want to wipe (e.g., `/dev/disk2`). 4. Wipe the disk securely using the command: 
1
diskutil eraseDisk /dev/disk2 GPT JHFS+ myDisk
**Using the `hdiutil` Command:** 1. Open Terminal. 2. Identify the disk you want to wipe using the command: 
1
hdiutil list
3. Unmount the disk if it's mounted using the command: 
1
hdiutil detach /dev/disk2s1
4. Wipe the disk securely using the command: 
1
sudo hdiutil zerofill /dev/disk2
**Additional Parameters for Secure Wiping:** * **`-passcount`:** Sets the number of passes for secure wiping (default: 1). * **`-secure`:** Performs secure wiping with random data instead of zeros. * **`-verbose`:** Outputs progress information during the wiping process. **Example:** To wipe the disk identified as `/dev/disk2` with 3 passes of secure wiping, use the following command: 
1
diskutil eraseDisk /dev/disk2 GPT JHFS+ myDisk -passcount 3 -secure -verbose
**Note:** * Ensure you have a backup of your data before wiping any disk. * It may take some time to wipe a disk securely, depending on its size.
