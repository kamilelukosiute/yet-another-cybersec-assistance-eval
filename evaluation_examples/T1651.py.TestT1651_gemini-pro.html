2. **Acquire Access Token:** Acquire an access token for the VM using an AAD user with permission to the VM. You can do this using the Get-AADAccessToken cmdlet: 
1
2

1

1
Run-AADRemoteScript -Resource "https://management.azure.com/" -AccessToken $AccessToken -ScriptPath "cmd.exe" -ArgumentList "dir"
**Note:** * The target VM must have PowerShell installed and remote script execution enabled. * The AAD user must have the appropriate RBAC permissions (e.g., Virtual Machine Administrator) on the VM. * For more information, refer to the AADInternals documentation: https://github.com/AzureAD/AzureADInternals/wiki/Run-Remote-Script-with-Azure-AD-Credentials

**Prerequisites:** * An Azure Virtual Machine (VM) with AADInternals installed. * A locally stored AADInternals binary. * An Azure Active Directory (AD) user account with sufficient permissions to run AADInternals. **Steps:** 1. **Connect to the VM:** - Using your AAD credentials, establish a remote desktop connection to the VM. 2. **Open a Command Prompt with Elevated Privileges:** - In the VM, press **Windows key + R** and type **"cmd"**. - Right-click on "Command Prompt" and select "Run as administrator". 3. **Copy AADInternals to the VM:** - Copy the AADInternals binary (e.g., "AADInternals.exe") from your local computer to the VM. - You can use the following command: 

1
2
     copy "C:\path\to\AADInternals.exe" "C:\destination\path\"
4. **Run AADInternals:** - Navigate to the directory where you copied AADInternals. - Run the following command: 
1
2
     AADInternals.exe -cmd "your_script" -target <target_user> -user <your_user> -password <your_password>
- Replace "your_script" with the script you want to run. - Replace "<target_user>" with the username of the target user you want to run the script as. - Replace "<your_user>" and "<your_password>" with your AAD credentials to authenticate the command. **Example:** To run a script called "test.ps1" as the user "UserA", you would run the following command: 
1
AADInternals.exe -cmd "test.ps1" -target UserA -user Administrator -password MyPassword
**Note:** * The "Administrator" user is required to run AADInternals with elevated privileges. * If the target user is not a local administrator, you may need to grant them local administrator permissions before running the script.
