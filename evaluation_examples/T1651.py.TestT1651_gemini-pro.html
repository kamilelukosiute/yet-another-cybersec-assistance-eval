1

1
2

1
2
3
4

How do I run a script on Azure VMs via AADInternals?

Output

**Prerequisites:** * Azure Subscription with AAD credentials * Application Registration with Service Principal in the Azure AD tenant * Azure VM created and running in the subscription **Steps:** 1. **Download AADInternals:** - Navigate to https://github.com/Azure/aad-pod-identity/releases/ and download the latest release compatible with your OS. 2. **Copy AADInternals to the VM:** - Use a tool like SFTP or SCP to copy `aad-pod-identity-*.tar.gz` to the VM. 3. **Extract AADInternals:** - On the VM, run the following command to extract the files: 

1
2
   tar -xf aad-pod-identity-*.tar.gz
4. **Establish AAD Connection:** - Run the following command to initialise the AAD connection using your service principal credentials: 
1
2
   aadpod-init --servicePrincipal <Application ID> --secret <Client secret>
- Provide your Application ID and Client secret from the service principal. 5. **Run the Script:** - Create a script on the VM in a file named `myscript.sh`. - For example: 
1
2
   echo Hello World!
- Run the script using the following command: 
1
2
   aadpod-run myscript.sh
**Additional Notes:** * To elevate the script to run as root, use `aadpod-run --privileged myscript.sh`. * You can pass arguments to the script by adding them after the script name. * The script runs in the context of AAD service principal, so the permissions of the service principal determine the script's capabilities. * To re-establish AAD connection if the credentials change, run `aadpod-rerun` before executing any more scripts.
