To ensure that your script runs automatically when a user logs into a Windows system, there are several methods you can use. These methods include leveraging built-in Windows features as well as scripting techniques to automate the setup. Here’s a detailed guide along with PowerShell scripts where applicable: ### 1. **Task Scheduler** #### Manual Setup 1. **Open Task Scheduler**: Press `Win + R`, type `taskschd.msc`, and press Enter. 2. **Create a New Task**: - Click on “Create Task…” in the Action pane. - In the General tab, provide a name and optionally a description for the task. - Select "Run only when user is logged on" or "Run whether user is logged on or not" depending on your needs. 3. **Set Trigger**: - Go to the Triggers tab and click "New...". - In the "Begin the task" dropdown, select "At log on". - Specify if it should apply for all users or a specific user. - Click OK. 4. **Set Action**: - Go to the Actions tab and click "New...". - In the "Action" dropdown, select "Start a program". - Browse to your script and select it. - Click OK. 5. **Finish**: Review your settings and click OK. #### Automate with PowerShell You can use the `New-ScheduledTask`, `New-ScheduledTaskAction`, and `New-ScheduledTaskTrigger` cmdlets to create the task programmatically:

Register-ScheduledTask -Action $action -Trigger $trigger -Principal $principal -TaskName "MyStartupScript"
### 2. **Startup Folder** #### Manual Setup 1. **Open Run Dialog**: Press `Win + R`, type `shell:startup`, and press Enter. 2. **Place a Shortcut in the Startup Folder**: - Copy your script or a shortcut to your script into this folder. - This will ensure that the script runs at login for the currently logged-in user. #### Automate with PowerShell You can use PowerShell to place a shortcut in the Startup folder:

### 3. **Registry Run Key** #### Manual Setup 1. **Open Registry Editor**: Press `Win + R`, type `regedit`, and press Enter. 2. **Navigate to the Run Key**: - For the current user: `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` - For all users: `HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run` 3. **Add a New String Value**: - Right-click in the pane and select New > String Value. - Set the name to whatever you want and set the value to your script path. #### Automate with PowerShell You can modify the registry with PowerShell:

Ensure that PowerShell scripts are run as an administrator when necessary. - **Error Handling**: Include error handling in your scripts to log errors or retry failed steps. - **Testing**: Always test automated scripts in a controlled environment before deploying widely. These methods offer a variety of options depending on your administrative access, script sensitivity, and the number of users the script needs to apply to. Each has its own benefits and application scenarios.
