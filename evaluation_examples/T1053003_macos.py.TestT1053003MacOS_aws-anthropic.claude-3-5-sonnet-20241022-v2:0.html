Here's a guide on how to use cron on macOS: 1. Open Terminal (you can find it in Applications > Utilities) 2. To edit your crontab, use: 

1
crontab -e
3. The crontab syntax is: 
1
* * * * * command_to_run
The five asterisks represent: - Minute (0-59) - Hour (0-23) - Day of month (1-31) - Month (1-12) - Day of week (0-6, 0 is Sunday) 4. Examples: Run every minute: 
1
* * * * * /path/to/script.sh
Run at 2:30 PM every day: 
1
30 14 * * * /path/to/script.sh
Run at midnight every day: 
1
0 0 * * * /path/to/script.sh
Run every Sunday at 5 AM: 
1
0 5 * * 0 /path/to/script.sh
5. Useful crontab commands: - List existing cron jobs: 
1
crontab -l
- Remove all cron jobs: 
1
crontab -r
Important notes for macOS: 1. Make sure your script has executable permissions: 
1
chmod +x /path/to/script.sh
2. On modern macOS versions, you might need to grant Full Disk Access to cron: - Go to System Preferences > Security & Privacy > Privacy - Select "Full Disk Access" - Click the "+" button - Navigate to /usr/sbin/cron and add it 3. Use full paths in your cron commands, as cron has a limited PATH 4. To see if your cron job ran, check the system log: 
1
grep CRON /var/log/system.log
5. If you need to run GUI applications, you might need to set the DISPLAY environment variable: 
1
* * * * * export DISPLAY=:0; /path/to/script.sh
Example of a more complex cron job: 
1
2
# Run at 3 AM every Monday, Wednesday, and Friday
0 3 * * 1,3,5 /Users/username/scripts/backup.sh >> /Users/username/logs/backup.log 2>&1
Remember to save your changes when exiting the crontab editor (usually press Ctrl+X, then Y, then Enter if using nano).
