Leveraging Windows system logs to access and export security event information is a powerful way to monitor and manage the security posture of a Windows system. Windows provides several built-in tools to facilitate this, including Event Viewer, PowerShell, and the Command Prompt. Below are detailed steps on how you can access and export security event information using these built-in utilities. ### Using Event Viewer: 1. **Open Event Viewer:** - Press `Win + R`, type `eventvwr.msc`, and press `Enter`. This opens the Event Viewer. 2. **Navigate to Security Logs:** - In the Event Viewer window, expand **"Windows Logs"** in the left-hand pane. - Click on **"Security"** to view security-related events. 3. **Filter Events:** - Right-click on the **Security logs** and select **"Filter Current Log..."**. - Use the filter to specify the event IDs or date range you are interested in. Common security event IDs include: - 4624: An account was successfully logged on - 4625: An account failed to log on - 4720: A user account was created - 4726: A user account was deleted - Configure other filter options as necessary, then click **"OK"**. 4. **Export Logs:** - Click on **"Action"** in the top menu, then select **"Save Filtered Log File As..."**. - Choose a location and a format (e.g., .evtx or .xml) to save the log file. ### Using PowerShell: 1. **Open PowerShell:** - Press `Win + X` and select **"Windows PowerShell"** from the menu. 2. **Retrieve Security Logs:** - Use the `Get-WinEvent` cmdlet to query security logs. For example, to get the most recent 100 security log events, you can use: 

1
2
     Get-WinEvent -LogName Security -MaxEvents 100
3. **Filter Events by ID or Other Criteria:** - To filter events by ID, for example, to only show failed logon attempts (ID 4625), use: 
1
2
     Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625}
4. **Export Logs to CSV:** - Combine the filtering with `Export-Csv` to save the data to a file: 
1
2
     Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} | Export-Csv -Path C:\Path\To\SecurityEvents.csv -NoTypeInformation
### Using the Command Prompt (Wevtutil): 1. **Open Command Prompt:** - Press `Win + R`, type `cmd`, and press `Enter`. 2. **List all Logs (Optional):** - Use the following command to see all available logs: 
1
2
     wevtutil el
3. **Export Security Logs:** - To export security logs to an .evtx file, use: 
1
2
     wevtutil epl Security C:\Path\To\SecurityLog.evtx
4. **Query Specific Events (Optional):** - You can query specific logs or events in XML format using: 
1
2
     wevtutil qe Security /f:xml > C:\Path\To\SecurityLog.xml
By using these built-in Windows tools, you can effectively access, filter, and export security event logs, which is crucial for maintaining the security of your Windows environment. Adjust the filters and export methods based on your specific operational and security requirements.
