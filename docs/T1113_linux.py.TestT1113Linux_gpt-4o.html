#!/bin/bash

# Directory to save screenshots
SAVE_DIR="$HOME/screenshots"

# Interval in seconds (e.g., 300 seconds for 5 minutes)
INTERVAL=300

# Ensure the SAVE_DIR exists
mkdir -p "$SAVE_DIR"

# Function to take a screenshot
take_screenshot() {
    local TIMESTAMP=$(date +"%Y-%m-%d_%H-%M-%S")
    local FILENAME="$SAVE_DIR/screenshot_$TIMESTAMP.png"
    scrot "$FILENAME"
}

# Run in a loop indefinitely
while true
do
    take_screenshot
    sleep $INTERVAL
done &  # The ampersand '&' allows it to run in the background

# To detach the script completely from the terminal, use 'nohup':
# nohup ./screenshot_script.sh &

# Inform the user
echo "Screenshot script is running in the background. Screenshots are saved to $SAVE_DIR every $INTERVAL seconds."
### Instructions: 1. **Save the Script**: Save the above script to a file, for example, `screenshot_script.sh`. 2. **Make It Executable**: Run `chmod +x screenshot_script.sh` to make the script executable. 3. **Run the Script**: Start the script using `./screenshot_script.sh` or run it in the background with `nohup` to detach it from the terminal completely: `nohup ./screenshot_script.sh &`. ### Notes: - **Script Directory**: The script saves screenshots in `~/screenshots`. You can change `SAVE_DIR` to another location if desired. - **Running Silently**: The script runs in the background due to the trailing `&` after the `done` loop. Using `nohup` ensures it continues running even if the terminal is closed. - **Adjusting the Interval**: Modify the `INTERVAL` variable to change the frequency of screenshots. - **Stopping the Script**: You'll need to find the PID of the script running in the background (`ps aux | grep screenshot_script`) and then kill it using `kill [PID]` if you want to stop it later on.
