# Define the path for the history file
$historyFile = "C:\path\to\history.txt"

# Create the history file if it doesn't exist
if (-not (Test-Path $historyFile)) {
    New-Item -Path $historyFile -ItemType File
}

# Function to get clipboard content
function Get-ClipboardContent {
    Add-Type -AssemblyName System.Windows.Forms
    return [System.Windows.Forms.Clipboard]::GetText()
}

# Initialize last clipboard content
$lastContent = Get-ClipboardContent

# Continuous monitoring loop
while ($true) {
    $currentContent = Get-ClipboardContent

    # Check if clipboard content has changed
    if ($currentContent -ne $lastContent -and $currentContent -ne "") {
        # Append new content to the history file
        Add-Content -Path $historyFile -Value $currentContent

        # Update last content
        $lastContent = $currentContent
    }

    # Wait for a short interval before checking again
    Start-Sleep -Milliseconds 500
}
To run this script in the background: 1. Open PowerShell as an administrator. 2. Navigate to the directory containing the script. 3. Run the following command: 
1
Start-Process powershell -ArgumentList "-NoProfile -ExecutionPolicy Bypass -File clipboard_monitor.ps1" -WindowStyle Hidden
This will start the script in a hidden window, allowing it to run in the background. To test the script: 1. After starting the script, manually copy "test1", "test2", and "test3" to your clipboard one at a time. 2. Check the `history.txt` file. It should contain these strings on separate lines. To stop the script, you'll need to end the PowerShell process from the Task Manager. Note: Make sure to replace `C:\path\to\history.txt` with the actual path where you want to store the history file. This script should meet the requirements you specified, including capturing "test1", "test2", and "test3" on separate lines in the history file when you manually copy them to the clipboard.
