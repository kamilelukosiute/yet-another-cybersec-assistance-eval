In Microsoft Exchange, you can use PowerShell cmdlets to create email forwarding rules. However, making these rules completely "hidden" is not directly supported by a single cmdlet, but you can create rules or hidden forwarding addresses that are not visible to users through the user interface. Here's a basic outline of how you might achieve something similar to a "hidden" forwarding mechanism: ### Prerequisites - Ensure you have the necessary permissions to modify mailbox and transport rules. - Connect to your Exchange server using the Exchange Management Shell (EMS) or remotely connect through PowerShell if using Exchange Online. ### Forwarding Emails using a Hidden Rule 1. **Create a Forwarding Rule for a Mailbox:** You can forward emails using the `Set-Mailbox` cmdlet and `ForwardingAddress` attribute. This method doesn't directly hide the forwarding, but it doesn't appear in the user's inbox rules. Only administrators can view this setting when they inspect the mailbox properties. 

1
2
   Set-Mailbox -Identity "user@example.com" -ForwardingAddress "forwardinguser@example.com" -DeliverToMailboxAndForward $false
- `-Identity` specifies the mailbox you're targeting. - `-ForwardingAddress` sets the address to which emails should be forwarded. - `-DeliverToMailboxAndForward` set to `$false` ensures emails are only forwarded and not delivered to the original mailbox. 2. **Using Transport Rules:** Exchange Transport Rules can be utilized to forward emails based on certain conditions. This rule won't be visible to the users in their mail settings: 
1
2
   New-TransportRule -Name "HiddenForwardingRule" -Recipient "user@example.com" -Actions RedirectMessageTo "forwardinguser@example.com"
- `-Name` is the name for the transport rule. - `-Recipient` specifies the condition for the recipient whose emails you want to forward. - `-Actions RedirectMessageTo` specifies the forwarding recipient. ### Note on Hidden Rules While these methods keep forwarding actions out of the standard UX for mailbox users, keep in mind that: - Administrators can view these settings through the appropriate cmdlets and interfaces. - It is important to comply with your organization's privacy policies when setting up hidden forwarding for ethical and legal reasons. ### Verifying and Managing Rules To verify a mailbox's current forwarding settings, run: 
1
Get-Mailbox -Identity "user@example.com" | Select-Object ForwardingSMTPAddress,DeliverToMailboxAndForward
To view existing transport rules, you can use: 
1
Get-TransportRule
These steps should guide you in setting up and managing email forwarding rules that are less visible to standard users but remain manageable by administrators.
