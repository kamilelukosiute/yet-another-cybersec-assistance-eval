   $shortcutName = "MyScriptShortcut.lnk"
   $wshShell = New-Object -ComObject WScript.Shell
   $shortcut = $wshShell.CreateShortcut((Join-Path $startupFolder $shortcutName))
   $shortcut.TargetPath = $scriptPath
   $shortcut.Save()
### Method 2: Task Scheduler 1. **Manual Setup:** - Open Task Scheduler (`taskschd.msc`). - Click on "Create Basic Task...". - Follow the wizard to specify a trigger "At log on" and select your script as the action. 2. **Automate with PowerShell:**

   Register-ScheduledTask -Action $action -Trigger $trigger -Principal $principal -TaskName "MyScriptTask" -Description "Run my script at logon"
### Method 3: Registry Run Key 1. **Manual Setup:** - Open Registry Editor (`regedit`). - Navigate to `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`. - Create a new String Value, name it something descriptive, and set the value to the full path of your script. 2. **Automate with PowerShell:**

### Method 4: Group Policy (for domain environments) 1. **Manual Setup:** - Open the Group Policy Management Console (`gpmc.msc`). - Navigate to User Configuration -> Windows Settings -> Scripts (Logon/Logoff). - Double-click on "Logon", and then click "Add" to include your script. 2. **Automate with PowerShell:** - Unfortunately, configuring GPO scripts with PowerShell isn't straightforward without third-party tools or modules. Always scan and verify scripts' origin. - **PowerShell Execution Policy:** Ensure the execution policy allows your script to run (`Set-ExecutionPolicy`, use `-Scope` for current user or session). - **User Permissions:** Ensure users have the necessary permissions to edit these settings or add scheduled tasks. By utilizing these methods, you can efficiently automate the execution of a script upon user login on Windows systems.

Choose to trigger the task for "Any user" or a specific user. 4. **Actions Tab**: - Click "New…" and select "Start a program".
