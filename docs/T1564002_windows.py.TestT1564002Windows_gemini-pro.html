**Method 1: Disable Visibility** 1. Open the **Local Users and Groups** tool (lusrmgr.msc). 2. Right-click on the administrative account you want to hide and select **Properties**. 3. Go to the **Member Of** tab. 4. Click **Add**. 5. Enter "Protected Users" in the "Enter the object names to select" field and click **Check Names**. 6. Click **OK**. 7. Click **Apply** and **OK**. **Method 2: Hide Display Name** 1. Open the **Registry Editor** (regedit). 2. Navigate to the following key: 

1
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
3. Right-click on the Winlogon key and select **New** > **String Value**. 4. Name the new value "HideDisplayName" and set its value to **1**. 5. Click **OK**. 6. Restart the computer. **Method 3: Use Account Filter** 1. Open the **Group Policy Editor** (gpedit.msc). 2. Navigate to the following path: 
1
Computer Configuration\Windows Settings\Security Settings\Local Policies\Security Options
3. Double-click on the **Interactive logon: Do not display last user name in logon screen** policy. 4. Select **Enabled**. 5. Click **Apply** and **OK**. **Note:** * Method 1 works for both Windows 10/11 and Windows Server. * Methods 2 and 3 only work for Windows 10/11. * If you enable "Use account filter" (Method 3), it will hide all non-administrator accounts from the login screen, not just administrative accounts.

Under **Local Computer Policy**, navigate to: 

1

3. Double-click on the **Access this computer from the network** policy. 4. Click the **Add User or Group** button. 5. Type the name of the administrative account you want to hide and click **OK**. 6. Click **OK** again to save the changes and close the dialog boxes. **Method 3: Using PowerShell** 1. Open PowerShell as an administrator. 2. Run the following command: 
1
Set-LocalUser -Name "AdminAccount" -Enabled False
**Note:** Replace "AdminAccount" with the name of the administrative account you want to hide. **Verification:** * **Method 1 and 2:** Log in to the system as a non-administrative user and open the Users folder. The hidden administrative account should not be listed. * **Method 3:** Run the command `Get-LocalUser` to view the current status of the account. The `Enabled` field should be set to `False`.

Query

i'm creating lots of accounts for our windows systems, and i want to hide the administrative accounts from being listed to the primary user. how do i do that?

Output

**Method 1: Using Local Security Policy** 1. Open "Local Security Policy" by pressing **Windows Key** + **R** and typing **secpol.msc**. 2. Navigate to **Local Policies** > **Security Options**. 3. Double-click on the policy **"Hide accounts and membership of administrators in the Administrative Tools group"**. 4. Select **"Enabled"** and click **"OK"**. **Method 2: Using the Registry Editor** 1. Open the Registry Editor by pressing **Windows Key** + **R** and typing **regedit**. 2. Navigate to the following registry key: 

1
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\Explorer
3. Create a new DWORD value named **"HideAdministratorAccounts"**. 4. Set the value data to **1**. **Method 3: Using a Group Policy Object (GPO)** **Note:** This method requires a domain environment. 1. Open the Group Policy Management Editor by pressing **Windows Key** + **R** and typing **gpedit.msc**. 2. Navigate to **User Configuration** > **Administrative Templates** > **Windows Components** > **Windows Explorer**. 3. Double-click on the policy **"Hide Administrator accounts in the Administrator Tools group"**. 4. Select **"Enabled"** and click **"OK"**. **Verification:** To verify that the administrative accounts are hidden, log in as a non-administrative user and check the "Computer Management" control panel. You should no longer see any administrator accounts listed under "Local Users and Groups" > "Users".
