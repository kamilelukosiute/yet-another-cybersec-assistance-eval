Wiping a disk from the command line on a macOS machine can be done using the `diskutil` and `dd` commands. Type the following command and press Enter: 

1
2
   diskutil list

1
diskutil unmountDisk /dev/diskX
Replace `/dev/diskX` with your actual disk identifier. ### Step 3: Wipe the Disk You can use either `diskutil secureErase` for a secure wipe or `dd` for a general wipe. #### Option 1: Using `diskutil secureErase` 1. Use the following command to securely erase the disk. 1
2
   diskutil secureErase 0 /dev/diskX
#### Option 2: Using `dd` 1. To write zeros over the entire disk: 
1
2
   sudo dd if=/dev/zero of=/dev/diskX bs=1m
2. To write random data (optional, for better security): 
1
2
   sudo dd if=/dev/urandom of=/dev/diskX bs=1m

1
diskutil list
### Important Considerations - **Data Loss Warning**: Both methods will permanently erase all data on the target disk. Make sure you have backups if needed. - **Admin Privileges**: You'll need administrative privileges to run these commands. You may be prompted to enter your password. - **Incorrect Disk Choice**: If you choose the wrong disk, you could erase important data. Double-check the disk identifier before proceeding. By following these steps, you can securely wipe a disk on macOS from the command line.

2

Replace `'DiskName'` with the desired name for the disk, and `diskX` with the actual disk identifier. 4. **Securely Erase Free Space (Optional):** If you want to securely erase the existing free space or the entire disk to ensure that data recovery is not possible, use the `secureErase` option. 2

Wiping a disk on a macOS machine can be done using the command line tool `diskutil`. This operation involves erasing all data on the disk, so it's crucial to ensure that you’ve backed up any important information before proceeding. Here’s a step-by-step guide: 1. **Open Terminal:** You can find Terminal in Applications > Utilities, or by searching for it in Spotlight (Cmd + Space, then type "Terminal"). 2. **List Available Disks:** Enter the following command to list all connected disks and their identifiers: 

1
2
   diskutil list
This command provides information on all the disks available, allowing you to identify which disk you want to wipe. Look for a line corresponding to the disk you want to erase, usually in the format `/dev/diskX`, where `X` is the disk number. 3. **Unmount the Disk (if necessary):** If the disk is currently mounted, you may need to unmount it before wiping. Replace `diskX` with the correct disk identifier: 
1
2
   diskutil unmountDisk /dev/diskX
4. **Erase the Disk:** Now, use the `diskutil` command to erase the disk. You can specify the file system format and a new name for the disk. The example below uses `JHFS+` (Journaled HFS Plus, Mac OS Extended) as the file system format, but you can choose other formats like `APFS` for newer Mac systems. Replace `diskX` with your actual disk identifier: 
1
2
   diskutil eraseDisk JHFS+ DiskName /dev/diskX
- `JHFS+` is the file system format. - `DiskName` is the new name you want for this disk. - `/dev/diskX` is the disk identifier. 5. **Securely Erase the Disk (Optional):** If you require a secure erase (overwriting the data to prevent recovery), you can use the `secureErase` option instead. The level of security ranges from `0` to `4`: 
1
2
   diskutil secureErase 0 /dev/diskX  # Single-pass (fastest)
The levels are: - `0`: Single-pass, writes zeros - `1`: Single-pass, writes random data - `2`: US DoD 7-pass secure erase - `3`: Gutmann algorithm, 35-pass - `4`: US DoE standard, 3-pass **Important**: Securely erasing can take a long time depending on the size of the drive and the level of security you choose. 6. **Verify:** Once the process is complete, you can verify by using `diskutil list` again to ensure the disk has been erased and is now appearing in the desired format. Remember, these actions will irreversibly erase all contents on the specified disk. Ensure you have selected the correct disk by double-checking its identifier, and confirm that you have backups of any important data before proceeding.
