Wiping a disk from the command line on a macOS machine can be done using the `diskutil` and `dd` commands. Hereâ€™s a step-by-step guide: ### Using `diskutil` to Erase a Disk 1. **Open Terminal**: You can find Terminal in `Applications > Utilities` or by searching for it using Spotlight (`Cmd + Space`, then type "Terminal"). 2. **List Disks**: Before erasing, identify the disk you want to wipe. Use the following command to list all available disks: 

1
2
   diskutil list
This will display all disks and their partitions. Look for the identifier of the disk you want to erase (e.g., `/dev/disk2`). 3. **Unmount the Disk** (if necessary): If the disk is mounted and in use, you'll need to unmount it first: 
1
2
   diskutil unmountDisk /dev/diskX
Replace `/dev/diskX` with the actual disk identifier. 4. **Erase the Disk**: Use the `diskutil eraseDisk` command to wipe the disk. Choose a format and a name for the disk. Common formats include `APFS` (for newer macOS installations) or `HFS+J` (Mac OS Extended Journaled). 
1
2
   diskutil eraseDisk APFS "NewDiskName" /dev/diskX
Replace `NewDiskName` with the desired name of the disk and `/dev/diskX` with the actual disk identifier. ### Using `dd` for Low-Level Overwrite For a more thorough disk wipe, you can use the `dd` command to overwrite the disk with zeroes or random data: 1. **Ensure Disk is Unmounted**: As mentioned, make sure the disk is not mounted using the `diskutil unmountDisk` command. 2. **Overwrite Using `dd`**: Use the following command to overwrite the entire disk with zeroes: 
1
2
   sudo dd if=/dev/zero of=/dev/diskX bs=1m
Or, for more security, overwrite with random data: 
1
2
   sudo dd if=/dev/urandom of=/dev/diskX bs=1m
Replace `/dev/diskX` with your target disk identifier. This process can take a long time depending on the size of the disk. ### Important Notes: - **Data Loss**: These actions will permanently erase all data on the drive. Make sure you have backed up any important data before proceeding. - **Disk Identification**: Be absolutely certain about the disk identifier; choosing the wrong disk can result in irreversible data loss on the wrong drive. - **Timing**: The `dd` method, especially with `/dev/urandom`, takes significantly longer than using `diskutil` with filesystem format, as it writes directly to every block on the disk. Proceed carefully and ensure you're working on the correct disk to avoid accidental data loss.
