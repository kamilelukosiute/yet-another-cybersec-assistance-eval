**Leveraging macOS System Logs for Security Event Information** macOS maintains extensive system logs that capture various system-related events, including security-relevant occurrences. You can utilize these logs to access and export security event information using built-in macOS binaries: 1. **Identify Security-Related Logs:** The primary macOS system log file is `/var/log/system.log`. However, for security-specific events, it's recommended to focus on the following additional log files: - `/private/var/log/security.log` - `/private/var/log/auth.log` - `/private/var/log/appfirewall.log` - `/private/var/log/audit.log` (if auditd is enabled) 2. **Access the Logs:** - **Using Terminal:** Open Terminal and navigate to the desired log file directory, typically `/var/log` or `/private/var/log`. Use the `tail` command to view the last portion of the log: 

1
2
3
     tail -n 100 /var/log/security.log
     tail -n 100 /private/var/log/auth.log
- **Using Console.app:** Open Console.app located in `/Applications/Utilities/`. In the left sidebar, select "System Logs" and then expand the "syslog" or "private" directories to access the desired log files. Filter the events based on the keyword "security" or relevant search terms. 3. **Export Security Logs:** - **Using log command:** This command allows you to extract specific log lines matching a pattern. For example, to export all security-related events from `/var/log/system.log`, use: 
1
2
     log show --predicate 'category == "security"' --info > security.log
- **Using Console.app:** In Console.app, select the events you want to export. Right-click and choose "Export" to save the events to a file in various formats such as Plain Text, JSON, or XML. **Tips:** - **Enable Auditd:** Auditd is a powerful auditing daemon that can collect and log detailed security events. Enable it using the following command: 
1
2
   sudo launchctl load -w /System/Library/LaunchDaemons/com.apple.auditd.plist
- **Configure Remote Logging:** You can configure macOS to send logs to a remote syslog server using the steps outlined in Apple's support article: [Configure remote logging](https://support.apple.com/en-us/HT200108). By following these steps, you can leverage macOS system logs to access and export valuable security event information that can aid in incident response and security monitoring efforts.

**Leveraging macOS System Logs to Access and Export Security Event Information with Built-in Binaries** **1. Collect Logs with `log show`** 

1
log show --style syslog --predicate 'subsystem == "securityd"'
* This command displays logs from the `securityd` subsystem in syslog format. * Filter logs by specific categories using `--predicate`, e.g., `authd` or `authorization`. **2. Extract Event Data with `auditlogfmt`** 
1
auditlogfmt -e -d 'event.code=="0","event.ident=="login"'
* Converts raw log data into human-readable format. * Uses `-e` to display events as JSON and `-d` to display only specific data (e.g., events with code "0" and ident "login"). **3. Filter Logs with `grep`** 
1
log show --predicate 'subsystem == "securityd"' | grep audit_succeeded
* Filters logs for specific keywords (here, "audit_succeeded") after collecting logs. **4. Parse Events with `auditp`** 
1
auditp -f TEXT -P date,uid,success,event,ticket 6091843248.903651.txt
* Parses audit events into a specified format (here, CSV). * Specify the input file (`6091843248.903651.txt`) which contains log data collected using `log show`. **5. Export Logs** To export logs to a file: 
1
log show --style syslog --predicate 'subsystem == "securityd"' > ~/security_events.log
To export in JSON format: 
1
log show --style json --predicate 'category == "securityd"' > ~/security_events.json
