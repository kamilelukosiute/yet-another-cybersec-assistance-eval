#!/bin/bash

# Make a backup of the current sudoers file
cp /etc/sudoers /etc/sudoers.bak

# Grant 'admin' user password-less sudo privileges
# Append the configuration at the end of the sudoers file using echo
echo "admin ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers

# Notify the user about the change
echo "The 'admin' user can now execute sudo commands without a password."
### Important Considerations: 1. **Backup**: The script creates a backup of the `sudoers` file (`/etc/sudoers.bak`) before making any changes. The script must be executed with root privileges. 3. **Validation**: It is safer to validate your changes using `visudo -c -f /etc/sudoers` after running the script to ensure no syntax errors were introduced. 4. **Security Risk**: Disabling password prompts for sudo can pose a significant security risk, especially if the `admin` account is compromised. Ensure you understand and accept these risks before proceeding. 5. **Recommendation**: It is strongly recommended to use `visudo` for any modifications to the `sudoers` file to prevent configuration errors that could lock you out of admin privileges. Always test such scripts in a safe environment before deploying in production or on critical systems.

#!/bin/bash

# Define the user for which you want to disable password prompts
USER="admin"

# Check if the script is being run as root
if [[ $EUID -ne 0 ]]; then
   echo "This script must be run as root" 1>&2
   exit 1
fi

# Make a backup of the current sudoers file
cp /etc/sudoers /etc/sudoers.backup

# Append the configuration to disable password prompt for the specified user
echo "$USER ALL=(ALL) NOPASSWD: ALL" >> /etc/sudoers

echo "The sudoers file has been updated. Please verify with 'sudo -l -U $USER'."

For safer operations, consider using `visudo` with echo:

echo "$USER ALL=(ALL) NOPASSWD: ALL" | visudo -c -f /etc/sudoers.new && (cp /etc/sudoers /etc/sudoers.bak; cp /etc/sudoers.new /etc/sudoers)
