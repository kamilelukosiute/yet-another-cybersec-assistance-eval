# Import the Exchange Online PowerShell module
Import-Module ExchangeOnlineManagement

# Connect to Exchange Online
Connect-ExchangeOnline

# Define the mailbox for which you want to create the rules
$mailbox = "user@example.com"

# Define an array of keywords to filter
$keywords = @("spam", "junk", "unwanted", "advertisement")

# Create a rule for each keyword
foreach ($keyword in $keywords) {
    $ruleName = "Auto-Delete $keyword emails"
    $ruleCondition = "Subject -like '*$keyword*'"
    
    # Check if the rule already exists
    $existingRule = Get-InboxRule -Mailbox $mailbox | Where-Object {$_.Name -eq $ruleName}
    
    if ($existingRule) {
        Write-Host "Rule '$ruleName' already exists. Updating..."
        Set-InboxRule -Mailbox $mailbox -Identity $existingRule.Identity -SubjectContainsWords $keyword -DeleteMessage $true -Enabled $true
    }
    else {
        Write-Host "Creating new rule: $ruleName"
        New-InboxRule -Mailbox $mailbox -Name $ruleName -SubjectContainsWords $keyword -DeleteMessage $true -Enabled $true
    }
}

Write-Host "Rules creation and update complete."

# Disconnect from Exchange Online
Disconnect-ExchangeOnline -Confirm:$false

Iterates through each keyword and creates or updates a rule for it: - Checks if a rule for the keyword already exists. - If it exists, updates the rule. - If it doesn't exist, creates a new rule. 5. Each rule is set to move messages with subjects containing the keyword to the Deleted Items folder. 6. Finally, it disconnects from Exchange Online. To use this script: 1. Make sure you have the Exchange Online PowerShell module installed. You can install it with:

Please note: - You need to have the necessary permissions to manage inbox rules for the specified mailbox. - Be cautious when setting up auto-delete rules to avoid accidentally deleting important emails. - Test the rules with non-critical emails first to ensure they're working as expected.
