Query

Output

To add `/new/dir` to the `PATH` variable in a way that prioritizes it (i.e., looks for executables in that directory before any others), you need to prepend it to the current `PATH`. Moreover, to make this change permanent in your bash shell, you'll need to modify your shell's configuration file, such as `~/.bashrc` or `~/.bash_profile`. Here's how you can do it: 1. **Edit the Configuration File:** - Open your `.bashrc` or `.bash_profile` file in a text editor. Which one you use might depend on your system; `.bashrc` is a common choice for many Linux distributions, while `.bash_profile` is often used on macOS. 

1
2
   nano ~/.bashrc
or 
1
2
   nano ~/.bash_profile
2. **Prepend `/new/dir` to the PATH:** Add the following line to the end of the file: 
1
2
   export PATH="/new/dir:$PATH"
This line adds `/new/dir` to the beginning of your `PATH` variable. 3. **Save and Exit:** Save the changes and exit the text editor. If you're using `nano`, you can usually do this by pressing `CTRL + X`, then `Y` to confirm changes, and `Enter` to exit. 4. **Apply the Changes:** To apply the changes immediately without needing to log out and back in, you can source the file: 
1
2
   source ~/.bashrc
or 
1
2
   source ~/.bash_profile
By doing this, you're ensuring that `/new/dir` is checked first for any executable files when you run commands, and the change will persist across terminal sessions.
