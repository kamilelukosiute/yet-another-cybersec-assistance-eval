Wiping a disk from the command line in Linux can be achieved using several tools, but one of the most common methods is using the `dd` command. This command allows you to overwrite the data on a disk with zeros (or another pattern), effectively wiping it. Here's how you can do it: ### Warning: Before proceeding, ensure you have selected the correct disk to wipe. This process will irreversibly erase all data on the specified disk. Double-check the disk identifier (e.g., `/dev/sdX`) using tools like `lsblk` or `fdisk -l` to avoid data loss on the wrong device. ### Steps to Wipe a Disk: 1. **Identify the Disk:** Open the terminal and use the following command to list all disks and their partitions: 

1
2
   lsblk
Or use: 
1
2
   sudo fdisk -l
Identify the target disk you want to wipe. It will be something like `/dev/sdX`, where `X` is a letter representing the disk. 2. **Use the `dd` Command to Wipe the Disk:** To overwrite the entire disk with zeros, use: 
1
2
   sudo dd if=/dev/zero of=/dev/sdX bs=4M status=progress
- `if=/dev/zero`: Uses a stream of zero bytes as the input. - `of=/dev/sdX`: Specifies the output file, the disk you want to wipe. - `bs=4M`: Sets the block size to 4 megabytes for faster processing. - `status=progress`: Provides progress updates during the operation. 3. **Verify Completion:** Once the process is completed, you'll return to the command prompt. No further steps are needed for a basic wipe. ### Additional Methods: - **Shred:** Another tool you can use is `shred`, which can overwrite the disk multiple times for added security: 
1
2
  sudo shred -n 3 -vz /dev/sdX
- `-n 3`: Overwrites the disk three times. - `-v`: Verbose mode, shows progress. - `-z`: Adds a final overwrite with zeros to hide the shredding. - **Wipe (Using Secure Wipe Tool):** If more thorough sanitization is needed, `wipe` provides options for multiple, more secure overwriting patterns: Install the `wipe` tool (if not already installed): 
1
2
  sudo apt install wipe
Then run: 
1
2
  sudo wipe /dev/sdX
Each of these methods has security trade-offs. For most practical purposes, overwriting with zeroes (`dd`) is sufficient, but for higher security needs, multiple passes with different patterns (`shred` or `wipe`) may be appropriate. ### Final Note: Make sure to replace `/dev/sdX` with the actual device identifier of your disk. This is crucial to prevent data loss on unintended devices.
