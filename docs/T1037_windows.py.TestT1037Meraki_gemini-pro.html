## Automating Script Execution at User Login **Method 1: Windows Task Scheduler** **Steps:** 1. Open Task Scheduler (press Win+R and type "taskschd.msc"). 2. Click "Create Basic Task" in the right-hand pane. 3. Enter a task name and description. 4. Select "Log on" as the trigger, then click "Next". 5. Choose the specific user or group whose login should trigger the task. 6. Configure the action to run the desired script. 7. Click "Finish" to create the task. **Method 2: Group Policy (Pro Edition Only)** **Steps:** 1. Open Group Policy Management (gpmc.msc). 2. Navigate to "Computer Configuration\Policies\Windows Settings\Scripts (Startup/Shutdown)". 3. Right-click and select "New" > "Logon". 4. Configure the settings as desired: - **Logon script:** Specify the script to run. - **User:** Choose the specific user or group. - **Run once per user login:** Ensures the script runs only once per login. 5. Click "OK" to save the settings. **Method 3: Registry Modification** **Steps (Caution: Modify the registry at your own risk):** 1. Open Registry Editor (regedit). 2. Navigate to "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon". 3. Right-click and create a new DWORD value named "AutoAdminLogon" (if it doesn't exist). 4. Set the value to "1". 5. Create a new string value named "LogonScript" (if it doesn't exist). 6. Set its value to the path of the script to run. 7. Click "OK" to save the changes. **Automating the Process with PowerShell** **PowerShell Script to Create Task Scheduler Task:**

### Automatic Script Execution on User Logon **Method 1: User Startup Folder** * **Manual Process:** 1. Create a shortcut to the script you want to run. 2. Copy the shortcut to `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`. * **PowerShell Script:** 

1
2
3
4
$user = Get-WmiObject -Class Win32_User | where { $_.name -eq 'Administrator' }
$shortcutPath = "$($user.HomeDirectory)\start menu\Programs\Startup\MyScript.lnk"
$linkPath = "$($user.HomeDirectory)\Desktop\MyScript.lnk"
New-Item -ItemType Shortcut -Path $shortcutPath -TargetPath $linkPath
**Method 2: Registry Startup Key** * **Manual Process:** 1. Open Registry Editor (`regedit`). 2. Navigate to `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`. 3. Create a new string value with the name of the script file. 4. Set the value data to the full path of the script. * **PowerShell Script:** 
1
2
$regKey = 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Run'
Set-ItemProperty -Path $regKey -Name "MyScript" -Value "$PWD\MyScript.ps1"
**Method 3: Task Scheduler** * **Manual Process:** 1. Open Task Scheduler. 2. Create a new task. 3. Select "Run a program" as the action. 4. Specify the path to the script in the "Program/script" field. 5. Set the task to trigger on user logon. * **PowerShell Script:** 
1
2
3
4
$taskName = "My Script"
$trigger = New-JobTrigger -AtLogon
$action = New-JobAction -Path ".\MyScript.ps1"
Register-ScheduledJob -Name $taskName -Trigger $trigger -Action $action
**Method 4: GPO (Group Policy Object)** * **Manual Process:** 1. Open the Group Policy Management Console (gpmc.msc). 2. Create a new GPO and link it to the desired OU. 3. Navigate to "User Configuration\Policies\Administrative Templates\System\Logon". 4. Enable "Run these programs at logon". 5. Add the path to the script in the "Programs" field. * **PowerShell Script:** WMI not recommended for automating GPO. **Method 5: Startup Services** * **Manual Process:** 1. Open Services Manager (`services.msc`). 2. Create a new service. 3. Set the service name and display name to the desired values. 4. Set the service to start automatically. 5. Specify the path to the script in the "Path to Executable" field. * **PowerShell Script:** Not recommended due to the low level access and complexity involved. **Additional Tips:** * Use a batch file or PowerShell script to wrap the script and handle any special requirements (e.g., permissions, arguments). * Test the script thoroughly before automating its execution. * Consider using a logging mechanism to track script execution and identify any potential issues.
