Output

Yes, you can use the `nmcli` command to enumerate the Wi-Fi profiles and passwords for each. Here's how: 1. Open a terminal window. 2. Run the following command: 

1
nmcli -t -f SSID,SECURITY,PSK connection show
This command will output a list of all the Wi-Fi profiles that are stored on your computer. The `SSID` column will show the name of the Wi-Fi network, the `SECURITY` column will show the type of security used by the network, and the `PSK` column will show the password for the network. If the `PSK` column is empty, then the network is not password-protected. Note: This command will only show the passwords for the Wi-Fi networks that you have previously connected to. If you have not connected to a Wi-Fi network before, then the `PSK` column will be empty.

Output

**Method 1: iwconfig** 

1
sudo iwconfig
This command will list the active Wi-Fi interfaces on your computer. Look for the interface that you're having trouble connecting to. It's often named `wlp3s0` or `wlan0`. Once you have identified the interface, run: 
1
sudo iwlist wlan0 scan | grep ESSID
This command will display the list of SSIDs (Wi-Fi network names) of all the available Wi-Fi networks. **Method 2: NetworkManager** If you have NetworkManager installed, you can use the following command: 
1
nmcli dev wifi list
This command will display a list of all the known Wi-Fi networks along with their SSIDs, signal strengths, and security settings. **Method 3: Root Access** If you have root access, you can use the following command to view the Wi-Fi passwords for all the known networks: 
1
sudo cat /etc/NetworkManager/system-connections/*.nmconnection | grep psk=
**Note:** * Be sure to replace `wlan0` in the command above with the correct interface name. * If the `psk=` field is empty, the network is using an unsecured connection or you may need to install the `wpa_supplicant` package and restart the NetworkManager service.
